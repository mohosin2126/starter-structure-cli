# Roadmap

`starter-structure-cli` is focused on making starter generation easier to discover, easier to maintain, and safer to publish.

## Short-Term Priorities

- expand the shipped starter catalog with stronger coverage across frontend, backend, fullstack, and monorepo combinations
- improve template matching so natural-language stack requests stay predictable
- strengthen template authoring ergonomics around reusable components and layered composition
- improve generated starter README quality and onboarding flow
- tighten validation around empty or incomplete generated templates

## Long-Term Direction

- support more reusable architectural layers without making presets difficult to reason about
- improve discoverability for template categories, aliases, and matching behavior
- provide stronger automation around template verification and package release workflows
- grow the starter library while keeping the repository structure understandable for contributors

## Maintainer Focus

The public package should stay:

- template-first
- easy to extend
- predictable from the CLI
- safe to publish from generated output

## How To Contribute

If you want to work on a roadmap item, open an issue or pull request with the proposed template category, stack combination, or workflow change so the repository structure stays coherent as it grows.
